Class-Objekt

Ein Class-Objekt repräsentiert eine Klassendefinition; es enthält statische Methoden und Eigenschaften.

Die folgende Dokumentation verwendet Cls als Platzhalter für jedes Class-Objekt. Beachten Sie, dass "class" nicht als Variablenname verwendet werden sollte, da es genutzt wird, um auf die Class-Klasse selbst zu verweisen. Normalerweise verweist man via Name auf ein Class-Objekt, der in dessen Klassendefinition angegeben ist.

Jedes Class-Objekt basiert auf dem, was auch immer es erweitert, oder auf Object, wenn es nicht angegeben ist. Object basiert auf Class.Prototype, das wiederum auf Object.Prototype basiert, so dass Klassen Methoden und Eigenschaften von jedem dieser Base-Objekte erben können.

"Statische" Methoden und Eigenschaften sind alle Methoden und Eigenschaften, die dem Class-Objekt selbst gehören (und daher nicht für eine bestimmte Instanz gelten), während Methoden und Eigenschaften für Instanzen der Klasse dem Prototyp der Klasse gehören.

Klassen haben typischerweise die folgenden Eigenschaften:

Methoden:

  • New

Eigenschaften:

Methoden

New

Konstruiert eine neue Instanz der Klasse.

Obj := Cls.New(Params*)

Diese statische Methode wird typischerweise von der Object-, Array- oder Map-Klasse geerbt. Es führt die folgenden Funktionen aus:

  • Speicher zuweisen und die binäre Struktur des Objekts initialisieren, was vom nativen Typ des Objekts abhängt (z. B. ob es ein Array oder Map-Objekt ist, oder nur ein Objekt).
  • Das Base des neuen Objekts auf Cls.Prototype setzen.
  • Die __Init-Methode des neuen Objekts aufrufen, falls sie eine hat. Diese Methode wird automatisch via Klassendefinitionen erzeugt; sie enthält alle im Klassenkörper definierten Instanzvariableninitialisierer.
  • Die __Init-Methode des neuen Objekts aufrufen, falls sie eine hat. Alle Parameter, die an New übergebenen werden, werden an __New weitergeleitet.
  • Das neue Objekt zurückgeben.

New kann innerhalb einer Klassendefinition überschrieben werden, indem eine statische Methode definiert wird, wie z. B. static New(). Dadurch können Klassen die Konstruktion neuer Instanzen ändern oder verhindern.

Beachten Sie, dass Class.New() genutzt werden kann, um ein neues Class-Objekt basierend auf Class.Prototype zu erstellen. Dieses neue Objekt hat jedoch zunächst keine New-Methode, da es keine Unterklasse von Object ist. Um es zu einer Unterklasse des Objekts zu machen, kann man es Base zuweisen, oder man kopiert die New-Methode mit GetMethod und DefineMethod. Ein Prototyp muss ebenfalls erstellt und der Klasse zugewiesen werden, bevor es mit der Standard-New-Methode instanziiert werden kann.

Eigenschaften

Prototype

Ermittelt oder setzt das Objekt, auf dem alle Instanzen der Klasse basieren.

Proto := Cls.Prototype
Cls.Prototype := Proto

Der Prototyp der Klasse enthält standardmäßig alle Instanzmethoden und dynamischen Eigenschaften, die in der Klassendefinition definiert sind, und kann genutzt werden, um Referenzen zu Methoden oder Eigenschaft-Getters/Setters abzurufen oder neue zu definieren. Das Skript kann zudem neue Werteigenschaften definieren, die als Standardeigenschaftswerte für alle Instanzen fungieren.

Prototype ist momentan definiert als eine eigene Eigenschaft eines Class-Objekts, das via Klassendefinition erzeugt wurde.
