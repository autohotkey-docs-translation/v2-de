Wie man AutoHotkey installiert

Falls noch nicht geschehen, können Sie AutoHotkey von einem der folgenden Orte herunterladen:

  • https://www.autohotkey.com/ - wenn Sie auf Download klicken, werden mehrere Optionen angezeigt.
  • https://www.autohotkey.com/download/ - die primären Download-Links sind für den aktuellen EXE-Installer, aber es gibt auch Links zur aktuellen ZIP-Datei und zum Archiv mit älteren Versionen.

Hinweis: Dieses Tutorial ist für AutoHotkey v2.

Der primäre Download-Link hat einen Dateinamen wie AutoHotkey_2.0_setup.exe. Führen Sie diese Datei aus, um die Installation von AutoHotkey zu starten.

Wenn Sie nicht der Administrator Ihres Computers sind, müssen Sie möglicherweise die Option Current user (aktueller Benutzer) wählen.

Andernfalls sind die empfohlenen Optionen bereits ausgefüllt, also klicken Sie einfach auf Install.

Für Benutzer von v1: AutoHotkey v2 enthält einen Launcher, der die parallele Verwendung mehrerer AutoHotkey-Versionen mit einer einzigen Dateiendung (.ahk) ermöglicht. Die Installation von AutoHotkey v1 und v2 in verschiedenen Verzeichnissen ist nicht erforderlich und wird derzeit nicht unterstützt.

Wenn Sie die Installation für alle Benutzer durchführen, wird eine Standard-UAC-Abfrage angezeigt, in der Sie Ihre Zustimmung als Administrator geben müssen (klicken Sie auf Ja).

Wenn keine Probleme aufgetreten sind, ist AutoHotkey nun installiert!

Sobald die Installation abgeschlossen ist, wird Dash automatisch angezeigt.

ZIP-Installer

AutoHotkey kann auch direkt aus der heruntergeladenen ZIP-Datei installiert werden.

  1. Öffnen Sie die ZIP-Datei mit dem Dateiexplorer (kein Entpacken erforderlich) oder entpacken Sie den Inhalt der ZIP-Datei in ein temporäres Verzeichnis.
  2. Führen Sie Install.cmd aus.
  3. Wenn Sie eine Meldung wie "Der Herausgeber konnte nicht verifiziert werden. Möchten Sie diese Software ausführen?" erhalten, klicken Sie auf Ausführen.
  4. Setzen Sie die Installation wie oben beschrieben fort.

Sicherheitsabfragen

Sie können eine oder mehrere Sicherheitsabfragen erhalten, die von verschiedenen Faktoren abhängen.

Webbrowser

Gängige Webbrowser zeigen möglicherweise eine Warnung wie "AutoHotkey_2.0_setup.exe wurde blockiert, weil es Ihr Gerät beschädigen könnte." Dies ist eine allgemeine Warnung, die für alle Arten von ausführbaren Dateien gilt, die noch nicht "häufig heruntergeladen" wurden. Dieses Problem tritt häufig bei neuen Softwareversionen auf und verschwindet wieder, wenn die Version von mehreren Benutzern heruntergeladen wurde.

Um den Download zu behalten, gibt es je nach Browser unterschiedliche Methoden. Suchen Sie nach einer Menüschaltfläche in der Nähe der Download-Anzeige oder versuchen Sie, einen Rechtsklick auf den blockierten Download zu machen.

Es kann vorkommen, dass der Download aufgrund einer Antivirus-Falschmeldung (False-Positive) blockiert wird; lesen Sie in diesem Fall den Abschnitt Antivirus.

Der Dienst "Google Safe Browsing" (der auch von anderen Browsern verwendet wird) ist dafür bekannt, falsche Warnungen bezüglich AutoHotkey anzuzeigen. Weitere Informationen finden Sie unter Safe Browsing.

SmartScreen

Microsoft Defender SmartScreen kann eine Meldung wie "Windows hat Ihren PC geschützt" anzeigen. Dies ist bei Software von Open-Source-Entwicklern und unabhängigen Softwareanbietern (Independent Software Vendors, kurz ISV) üblich, insbesondere kurz nach der Veröffentlichung einer neuen Version. Der folgende Blogeintrag von Louis Kessler beschreibt das Problem sehr gut: That’s not very Smart of you, Microsoft

Um mit der Installation fortzufahren, klicken Sie auf Weitere Informationen und dann auf Trotzdem ausführen.
