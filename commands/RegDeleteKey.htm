RegDeleteKey

Löscht einen Subkey aus der Registry.

RegDeleteKey KeyName

Parameter

KeyName

Typ: Zeichenkette

Der vollständige Name des Registry-Keys.

Der Name muss entweder mit HKEY_LOCAL_MACHINE, HKEY_USERS, HKEY_CURRENT_USER, HKEY_CLASSES_ROOT oder HKEY_CURRENT_CONFIG beginnen (oder jeweils die Abkürzung, wie z. B. HKLM). Um Zugriff auf eine Remote-Registry zu erhalten, setzen Sie wie folgt den Computernamen und einen Schrägstrich davor: \\workstation01\HKEY_LOCAL_MACHINE

KeyName kann nur weggelassen werden, wenn eine Registry-Schleife läuft - in diesem Fall wird standardmäßig der Key des aktuellen Schleifenelements verwendet. Wenn das Element ein Subkey ist, wird standardmäßig der vollständige Name dieses Subkeys benutzt.

Fehlerbehandlung

Bei Misserfolg wird ein OSError ausgelöst.

A_LastError wird auf das Ergebnis der Systemfunktion GetLastError() gesetzt.

Bemerkungen

Warnung: Das Löschen von Registry-Einträgen kann erhebliche Probleme verursachen - also seien Sie vorsichtig!

Mit der Registry-Schleife können mehrere Registry-Keys und -Werte abgerufen und bearbeitet werden.

Innerhalb einer Registry-Schleife löscht RegDeleteKey nicht unbedingt das aktuelle Schleifenelement. Wenn das Element ein Subkey ist, wird RegDeleteKey() den Key selbst löschen. Wenn das Element ein Wert ist, wird RegDeleteKey() den Key, der diesen Wert enthält, löschen, inklusive aller Subkeys und Werte.

In den Bemerkungen zur Registry-Schleife finden Sie Hinweise, wie man auf die Registry eines Remotecomputers zugreifen kann.

Mit SetRegView können Einträge aus den 64-Bit-Registry-Bereichen in einem 32-Bit-Skript oder umgekehrt entfernt werden.
