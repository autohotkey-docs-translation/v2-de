Catch

Bestimmt den Code, der ausgeführt werden soll, wenn ein Wert oder Fehler während einer Try-Anweisung ausgelöst wird.

Catch ErrorKlasse as AusgabeVar
{
    Anweisungen
}

Parameter

ErrorKlasse

Typ: Klasse

Die Klasse des Wertes, die abgefangen werden soll, z. B. Error, TimeoutError oder MeinEigenerError. Dies kann auch eine kommagetrennte Liste von Klassen sein. Klassen müssen mit ihrem genauen vollständigen Namen angegeben werden (also keine Ausdrücke), da der Prototyp jeder Klasse während der Ausführung des Skripts aufgelöst wird. Es können sowohl interne als auch benutzerdefinierte Klassen verwendet werden, sogar solche, die nicht von Error abgeleitet sind.

Wenn keine Klassen angegeben sind, wird standardmäßig Error verwendet.

Mit Catch Any kann alles abgefangen werden.

Es wird ein Ladezeitfehler angezeigt, wenn ein ungültiger Klassenname verwendet wird oder wenn der Zugriff auf eine Klasse aufgrund des Vorhandenseins einer lokalen Variable mit demselben Namen nicht möglich ist.

AusgabeVar

Typ: Variable

Die Variable, in der die Daten des via Throw ausgelösten Wertes gespeichert werden sollen, was typischerweise ein Error-Objekt ist. Dies kann keine dynamische Variable sein.

Lässt man diesen Parameter weg, ist der direkte Zugriff auf den via Throw ausgelösten Wert nicht möglich. Man könnte ihn aber mithilfe von Throw ohne Parameter erneut auslösen.

Anweisungen

Anweisungen, die ausgeführt werden sollen, wenn eine Ausnahme ausgelöst wird.

Geschweifte Klammern sind in der Regel nicht erforderlich, wenn nur eine einzige Anweisung verwendet wird. Weitere Informationen finden Sie unter {...} (Block).

Bemerkungen

Es können mehrere Catch-Anweisungen hintereinander verwendet werden, wobei jede eine andere Klasse (oder mehrere Klassen) angibt. Wenn der Wert keine Instanz einer der aufgelisteten Klassen ist, wird er nicht von diesem Try-Catch abgefangen, aber möglicherweise von einem weiter oben im Aufrufstapel.

Jedes Catch muss mit einer darüberliegenden Try-Anweisung verbunden werden. Ein Catch bezieht sich immer auf die nächst darüberliegende noch nicht beanspruchte Try-Anweisung, sofern dieses Verhalten nicht mit einem Block geändert wurde.

Die Parameterliste kann optional mit runden Klammern umschlossen werden - in diesem Fall ist das Leer- oder Tabulatorzeichen nach Catch optional.

Nach Catch kann optional ein Else erfolgen, das ausgeführt wird, wenn innerhalb des zugehörigen Try-Blocks keine Ausnahme ausgelöst wurde.

Der One True Brace Style (OTB) kann je nach Bedarf angewendet werden. Zum Beispiel:

try {
    ...
} catch Error {
    ...
}

Ladezeitfehler können nicht abgefangen werden, da sie vor dem Ausführen der Try-Anweisung auftreten.
