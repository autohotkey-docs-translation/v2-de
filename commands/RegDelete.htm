RegDelete

Löscht einen Wert aus der Registry.

RegDelete KeyName, Wertname

Parameter

KeyName

Typ: Zeichenkette

Der vollständige Name des Registry-Keys.

Der Name muss entweder mit HKEY_LOCAL_MACHINE, HKEY_USERS, HKEY_CURRENT_USER, HKEY_CLASSES_ROOT oder HKEY_CURRENT_CONFIG beginnen (oder jeweils die Abkürzung, wie z. B. HKLM). Um eine Remote-Registry anzusteuern, setzt man vor dem Rootkey den Computernamen und einen Schrägstrich. Zum Beispiel: \\workstation01\HKEY_LOCAL_MACHINE

KeyName kann nur weggelassen werden, wenn eine Registry-Schleife läuft - in diesem Fall wird standardmäßig der Key des aktuellen Schleifenelements verwendet. Wenn das Element ein Subkey ist, wird standardmäßig der vollständige Name dieses Subkeys benutzt. Wenn das Element ein Wert ist, gilt für Wertname standardmäßig der Name dieses Wertes, was aber überschrieben werden kann.

Wertname

Typ: Zeichenkette

Der Name des Wertes, der gelöscht werden soll. Wenn dieser Parameter weggelassen wird oder leer ist, wird der Standardwert des Keys gelöscht (außer wie oben erwähnt). Der Standardwert wird im RegEdit-Tool als "(Standard)" angezeigt.

Fehlerbehandlung

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

A_LastError wird auf das Ergebnis der Systemfunktion GetLastError() gesetzt.

Bemerkungen

Warnung: Das Löschen von Registry-Einträgen kann erhebliche Probleme verursachen - also seien Sie vorsichtig!

Mit der Registry-Schleife können mehrere Registry-Keys und -Werte abgerufen und bearbeitet werden.

Innerhalb einer Registry-Schleife löscht RegDelete nicht unbedingt das aktuelle Schleifenelement. Wenn das Element ein Subkey ist, wird RegDelete() nur seinen Standardwert löschen.

In den Bemerkungen zur Registry-Schleife finden Sie Hinweise, wie man auf die Registry eines Remotecomputers zugreifen kann.

Mit SetRegView können Einträge aus den 64-Bit-Registry-Bereichen in einem 32-Bit-Skript oder umgekehrt entfernt werden.
